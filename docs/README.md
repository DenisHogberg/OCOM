# OCOM Documentation

Welcome to the official documentation of the Object-Centric Operating Model (OCOM).

This directory contains the complete specification of the OCOM framework, organized by section.

## Contents

- [AI](AI/) — AI agents, context, evaluation, knowledge, prompts, and tools
- [Core](Core/) — Governance, manifest, modeling rules, naming, principles, terminology, versioning
- [Domains](Domains/) — Business domains (Affiliate, BI, CRM, Compliance, Finance, HR, Legal, Marketing, Operations, Payments, Product, Support)
- [Entities](Entities/) — Core business entities (Player, Payment, Affiliate, Wallet, Transaction, etc.)
- [Examples](Examples/) — Worked examples, including the iGaming reference scenario
- [Language](Language/) — Notation, syntax, schema, vocabulary, and conformance rules
- [Lifecycles](Lifecycles/) — Commercial, content, financial, operational, and organizational lifecycles
- [Memory](Memory/) — Layered memory, evidence overlay, retention, and write-back governance
- [Meta](Meta/) — Meta-model constructs (Object, Relationship, Capability, Policy, Contract, etc.)
- [Models](Models/) — Domain, entity, event, lifecycle, relationship, state, and workflow models
- [Reference Architecture](Reference%20Architecture/) — AI, business-event, domain, enterprise, and object architecture
- [Workflows](Workflows/) — Workflow specifications (planned, not yet populated)

Documents are currently maintained as Word (`.docx`) files pending conversion to Markdown.
