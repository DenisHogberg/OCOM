# Versioning

**Document ID:** Core-06

**Status:** Draft

**Version:** 0.1

**Last Updated:** 20 July 2026

---

# Purpose

This document defines the versioning policy for the specification.

---

# Version Format

The specification follows semantic versioning.

MAJOR.MINOR.PATCH

Examples:

1.0.0

1.2.0

2.1.3

---

# Major Version

Incremented when breaking conceptual changes are introduced.

---

# Minor Version

Incremented when new concepts are added while maintaining compatibility.

---

# Patch Version

Incremented when documentation, examples, or editorial corrections are introduced.

---

# Compatibility

Minor and Patch releases should remain backward compatible whenever possible.
